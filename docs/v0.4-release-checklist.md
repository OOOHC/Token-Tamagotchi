# v0.4 Release Engineering Checklist

## Goal

Prepare Token Tamagotchi for daily use outside the development loop without changing the core v0.3 companion experience.

## Scope

- [ ] Produce a Windows installer build.
- [ ] Document macOS and Linux build requirements.
- [ ] Add first-run onboarding for local Codex access and privacy boundaries.
- [ ] Add startup behavior settings.
- [ ] Add show/hide/refresh/quit controls through a tray or menu bar surface.
- [ ] Add a production-safe settings path for enabling debug tools locally.
- [ ] Verify window bounds on single monitor, multi-monitor, and display scaling setups.
- [ ] Verify light and dark desktop readability with release screenshots.
- [ ] Measure idle, drag, expanded panel, and details panel performance.
- [ ] Write release notes and installation instructions.

## Non-Goals

- Predictive task estimates.
- AI prompt recommendations.
- New provider adapters.
- Cloud sync, telemetry, or network proxying.

## Release Gate

- [ ] `npm run build`
- [ ] `cargo build`
- [ ] Installer launches on a clean Windows machine.
- [ ] Debug tools are hidden unless explicitly enabled.
- [ ] No private local paths, prompts, or account details appear in screenshots or docs.
