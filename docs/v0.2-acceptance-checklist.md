# v0.2 Acceptance Checklist

## Completed

- [x] App reads Codex's local app-server `account/rateLimits/read` response.
- [x] 5-hour food meter aligns to `100 - primary.usedPercent`.
- [x] 7-day food meter aligns to `100 - secondary.usedPercent`.
- [x] App auto-refreshes quota state every 30 seconds.
- [x] Details panel shows 5-hour used tokens, estimated total tokens, and reset time.
- [x] Details panel shows 7-day used tokens, estimated total tokens, and reset date/time.
- [x] Manual refresh is available.
- [x] UI shows checking, synced, and offline states.
- [x] Low 5-hour quota shows a local alert.
- [x] Manual paste/parser diagnostics are moved into a debug section.
- [x] Default mode shows only the live mood companion.
- [x] Companion mood updates from the configured quota thresholds.
- [x] Clicking the companion shows a temporary quota mood/status message bubble.
- [x] Food-meter toggle appears after interacting with the companion.
- [x] Side toggle button reveals mini window controls, 5-hour and 7-day food meters, and the Details button.
- [x] Companion surface can be dragged to reposition the floating window.
- [x] Companion window is constrained to the current screen after drag and layout changes.
- [x] Layout changes preserve the companion's apparent desktop position when expanding or collapsing panels.
- [x] Bubble, toggle button, food meters, and details use adaptive window layouts to stay visible near screen edges.
- [x] Expanded panels can appear above or below the companion depending on screen position.
- [x] Expanded details mode keeps mini window controls available.
- [x] Details content is scrollable inside the details panel without adding a scrollbar to the companion surface.
- [x] Companion has click feedback, mood-specific facial expression, and mood-specific idle animation.
